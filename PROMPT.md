# Ralph Loop: Генерация отчётов по темам ML/AI в горнодобыче

## Твоя задача

Ты генерируешь краткие исследовательские отчёты (300-500 слов) для тем применения ML/AI в горнодобывающей промышленности. Каждую итерацию ты обрабатываешь ОДНУ тему.

## Алгоритм каждой итерации

### Шаг 1: Найди следующую тему без отчёта

Прочитай файл `src/data/topics.js` — он содержит массив `topics` с полями `{ id, name, sectionId, companies: [{ name, url }] }`.

Затем проверь директорию `src/data/reports/` — какие файлы `{id}.json` уже существуют.

Найди первую тему (по id), для которой ещё нет файла отчёта.

Если ВСЕ темы уже имеют отчёты — выведи:

<promise>ALL REPORTS DONE</promise>

### Шаг 2: Исследуй тему

Используй скил `deep-research` для исследования темы. Режим: **quick** (3 фазы, минимальный).

Передай в исследование:
- Название темы (русское, из topics.js)
- Английский перевод темы для поиска
- Уже известные компании и их URL из данных карточки (companies массив)

ВАЖНО: deep-research скил генерирует полный отчёт — тебе нужно извлечь из него только ключевую информацию и переформатировать в короткий отчёт по шаблону ниже.

Если deep-research скил недоступен или не срабатывает — используй WebSearch напрямую (3-5 поисков по теме) и собери информацию вручную.

### Шаг 3: Сохрани отчёт

Сохрани результат в файл `src/data/reports/{topic_id}.json` строго по этой схеме:

```json
{
  "topicId": <number>,
  "title": "<краткое название темы на русском>",
  "summary": "<2-3 предложения простым языком: что это и зачем. Для менеджера, не для инженера.>",
  "howItWorks": "<Что на входе, что на выходе, какие методы ML. 3-5 предложений без формул.>",
  "companies": [
    {
      "name": "<компания>",
      "description": "<Что конкретно делает: продукт, результат, клиенты. 1-2 предложения.>",
      "url": "<ссылка-пруф>"
    }
  ],
  "businessEffect": "<Конкретные цифры: экономия X%, ускорение в Y раз, точность Z%. 1-3 факта.>",
  "maturityVerdict": "<Mainstream/Зрелая/Развивающаяся/Frontier — и почему, 1 предложение.>",
  "sources": ["<url1>", "<url2>", "..."]
}
```

### Требования к контенту

- **Язык:** русский
- **Объём:** 300-500 слов суммарно по всем текстовым полям
- **Компании:** используй данные из карточки (topics.js) как основу. Добавь детали из исследования. Если в карточке 2-3 компании, можешь добавить ещё из исследования.
- **Никаких выдуманных фактов** — только то, что подтверждено источниками
- **Бизнес-эффект:** конкретные цифры, не общие слова. Если цифр нет — напиши что конкретно даёт технология.
- **sources:** включи URL из карточки + новые найденные

### Шаг 4: Подтверди

Выведи: "Отчёт для темы {id} ({name}) сохранён. Обработано: X из 139."

## Важно

- Обрабатывай ОДНУ тему за итерацию
- Не пропускай темы — иди строго по порядку id
- Не перезаписывай существующие отчёты
- Если поиск не дал результатов для темы — всё равно напиши отчёт на основе общих знаний и данных карточки, но отметь в sources что источников мало
