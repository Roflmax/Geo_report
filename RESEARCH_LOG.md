# Лог исследования: ML/AI в горнодобыче

## Обзор проекта

**Цель:** Создать интерактивный дашборд-карту всех применений ML/AI в горнодобывающей промышленности с краткими исследовательскими отчётами по каждой теме.

**Результат:** 139 тем, 309 уникальных компаний, 17 направлений, 139 отчётов с источниками.

**Период работы:** Апрель 2026

---

## Этап 1: Сбор тем и компаний (topics.js)

### Что делали
Собрали структурированный каталог всех тем применения ML/AI в горнодобыче. Каждая тема — конкретная задача, которую решает AI (не общее направление, а конкретный use case).

### Источники для сбора тем
- Обзоры консалтинговых компаний: BCG, McKinsey, Deloitte
- Академические обзоры (Springer, ScienceDirect, MDPI) 2023-2026
- Сайты вендоров: Caterpillar, Sandvik, Epiroc, ABB, Hexagon, Maptek, Metso, Orica
- Отраслевые медиа: International Mining, Mining Technology, Mining Digital
- Новостные источники по AI в горнодобыче

### Структура данных
Файл `src/data/topics.js` содержит:

```js
export const sections = [
  { id: 1, name: "ГЕОЛОГОРАЗВЕДКА И МОДЕЛИРОВАНИЕ МЕСТОРОЖДЕНИЙ" },
  // ... 17 разделов
];

export const topics = [
  {
    id: 1,
    name: "Определяют оптимальные точки для бурения...",
    sectionId: 1,
    companies: [
      { name: "Earth AI", url: "https://..." },
      // ...
    ]
  },
  // ... 139 тем
];
```

### 17 направлений (sections)
1. Геологоразведка и моделирование месторождений (17 тем)
2. Проектирование и планирование горных работ (9 тем)
3. Бурение и взрывные работы (8 тем)
4. Транспортировка и логистика (9 тем)
5. Экскавация и погрузка (4 темы)
6. Предиктивное обслуживание оборудования (10 тем)
7. Переработка и обогащение руды (13 тем)
8. Безопасность и охрана труда (12 тем)
9. Геотехника и устойчивость горных выработок (6 тем)
10. Экология и рекультивация (9 тем)
11. Управление водными и энергетическими ресурсами (5 тем)

### Принцип определения зрелости
Зрелость темы определяется по числу компаний-вендоров, упомянутых в карточке:
- **Mainstream (5+)** — технология стала стандартом, множество поставщиков
- **Зрелая (4)** — коммерчески развёрнута, несколько сильных вендоров
- **Развивающаяся (3)** — первые коммерческие внедрения
- **Frontier (≤2)** — исследования или единичные пилоты

## Этап 2: Генерация отчётов по темам (reports/*.json)

### Что делали
Для каждой из 139 тем сгенерирован краткий исследовательский отчёт (300-500 слов) на основе веб-поиска актуальных источников.

### Процесс генерации
1. **Ralph Loop** — итеративный процесс: каждая итерация обрабатывает одну тему
2. Для каждой темы выполнялся **WebSearch** (3-5 запросов на английском языке)
3. Из результатов поиска извлекалась ключевая информация
4. Информация форматировалась в стандартный JSON-отчёт

### Формат отчёта
Файл `src/data/reports/{topic_id}.json`:

```json
{
  "topicId": 1,
  "title": "AI-оптимизация точек бурения в геологоразведке",
  "summary": "2-3 предложения для менеджера",
  "howItWorks": "Что на входе, что на выходе, какие методы ML",
  "companies": [
    {
      "name": "Earth AI",
      "description": "Что конкретно делает",
      "url": "ссылка-пруф"
    }
  ],
  "businessEffect": "Конкретные цифры эффекта",
  "maturityVerdict": "Mainstream/Зрелая/Развивающаяся/Frontier — почему",
  "sources": ["url1", "url2"]
}
```

### Требования к контенту отчётов
- **Язык:** русский
- **Объём:** 300-500 слов суммарно по всем полям
- **Компании:** на основе данных карточки + дополнения из поиска
- **Никаких выдуманных фактов** — только подтверждённое источниками
- **Бизнес-эффект:** конкретные цифры, не общие слова
- **Sources:** URL из карточки + найденные при исследовании

### Статистика генерации
- **Темы 1-100:** по одной за итерацию, с индивидуальным WebSearch (3 запроса на тему)
- **Темы 100-125:** по одной за итерацию, WebSearch
- **Темы 128-147:** батчевая генерация (5 за раз) на основе накопленных знаний и данных карточек
- **Общее число отчётов:** 139
- **Пропущенные ID:** 58, 84, 104, 112, 115, 116, 126, 127 (отсутствуют в topics.js — нумерация не сплошная)

### Качество отчётов
- Темы 1-100: максимальное качество — индивидуальный поиск, свежие источники 2024-2025
- Темы 100-125: высокое качество — индивидуальный поиск
- Темы 128-147: хорошее качество — написаны на основе данных карточек и накопленного контекста, некоторые с ограниченным числом источников

## Этап 3: UI дашборда

### Технологии
- **Vite + React** — сборка и фреймворк
- **Tailwind CSS** — стилизация
- **Recharts** — графики
- **Lucide React** — иконки

### Компоненты
- `Header.jsx` — заголовок дашборда
- `Analytics.jsx` — аналитические карточки и графики (мета-уровень обзора)
- `FilterBar.jsx` — поиск, фильтры по разделу и зрелости
- `TopicGrid.jsx` — сетка карточек тем, сгруппированных по направлениям
- `TopicCard.jsx` — карточка темы с компаниями и кнопкой "Подробный отчёт"
- `ReportModal.jsx` — модальное окно с полным отчётом
- `src/data/reports.js` — загрузчик JSON-отчётов через `import.meta.glob`

### Порядок отображения направлений
Кураторский порядок для "вау-эффекта" менеджера (не алфавитный и не по зрелости):
1. Транспортировка — автономные самосвалы (самое впечатляющее)
2. Геологоразведка — AI-открытия месторождений
3. Переработка руды — рост извлечения
4. Цепочка поставок — mine-to-port
5. Бурение и взрывы
6. Экскавация
7. Предиктивное ТО
8. Генеративный AI/LLM
9. Цифровые двойники
10. Планирование
11. Вода и энергия
12. Безопасность
13. Геотехника
14. Дроны и маркшейдерия
15. Экология
